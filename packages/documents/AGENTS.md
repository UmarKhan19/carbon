# @carbon/documents

PDF generation, email templates, ZPL labels, and QR/barcode utilities.

## Always

- **Style PDFs with `useTw()` (react-pdf-tailwind), not raw `StyleSheet`** — semantic grays (`gray-600` = heading, `gray-800` = body) keep themes consistent across all blocks
- **Resolve async work before render** — QR codes, barcodes, images, and fonts must be resolved to data URLs / registered before the synchronous component tree renders
- **Use `@bwip-js/node` for QR/barcodes** — there is no `qrcode` dependency; call `generateQRCode()` or `generateBarcode()` from `src/qr/`
- **Route fonts through `getSafeFontFamily()`** — every render path must use this (falls back to Helvetica) so unregistered families never throw
- **Register new block-driven docs in `pdf/preview-documents.tsx`** — `DOCUMENT_PDFS[type] = { Component, sample }` is required for the template preview route

## Ask First

- Adding a new document block **type** (coordinate with the template engine — see `document-template-customizer.md`)
- Changing font bundling (woff format, NOT woff2 — react-pdf can't handle woff2/variable fonts)
- Modifying the `Template.tsx` page shell or shared chrome components

## Never

- Call `generateQRCodeSync` — it throws "not supported"
- Use `renderToBuffer` — routes use `renderToStream`, drain to Buffer manually, wrap in `new Uint8Array(body)`
- Copy `@react-pdf/renderer` v4 examples — Carbon pins v3.x (different API)

## Validation Commands

```bash
pnpm --filter @carbon/documents test        # vitest
pnpm --filter @carbon/documents typecheck   # tsgo --noEmit
pnpm --filter @carbon/documents build       # generates fonts.data.ts
```

## Key Patterns

- **Two PDF styles**: block-registry driven (customizable) vs hand-built (fixed layout labels)
- **Block shape**: `({ block, data }) => JSX.Element | null`, keyed in per-doc `registry.tsx`
- **Fonts**: bundled as base64 woff in gitignored `pdf/fonts.data.ts`, generated by `scripts/generate-fonts.mjs`
- **Exports**: `./pdf`, `./email`, `./template`, `./zpl`, `./qr`, `./labels`
