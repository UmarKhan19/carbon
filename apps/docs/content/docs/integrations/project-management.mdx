When a quality problem needs engineering's attention, Carbon can push it into the issue tracker your team
already lives in. Both connectors are aimed at **quality (<Term id="nonconformance">nonconformance</Term>) issues**.

## Linear

Authenticate with a **Linear API key** (it begins with `lin_api`). Once connected, Carbon syncs issues to
Linear, keeping each Carbon issue matched to the same Linear issue.

| Setting | What it controls |
| --- | --- |
| API key | Your Linear API key — Carbon checks the `lin_api` prefix. |

## Jira

Connect over **OAuth** — there are no fields to fill in beyond authorizing Carbon. Once connected, you can
link a quality issue to a Jira issue.

<Callout type="warn">
Jira only appears when its OAuth client is configured server-side (`JIRA_CLIENT_ID`) — see
[Environment variables](/docs/platform/environment-variables).
</Callout>
