# Agent Knowledge Base (docs → agent)

The in-app agent answers doc questions from a **generated, committed copy** of the docs
site, not from `docs/content` directly (Vite can't glob across packages into `docs/`, and
the erp Docker image copies `apps/**` but **not** `docs/**`). The corpus lives at:

- `apps/erp/app/modules/agent/kb/<slug>.md` — one component-stripped markdown file per doc
- `apps/erp/app/modules/agent/kb/manifest.json` — the keyword index `search_docs` scans
- `apps/erp/app/modules/agent/kb/README.md` — auto-written "do not edit" notice

All of it is **generated by `scripts/generate-agent-kb.ts`** from `docs/content/**` and is
**committed** (tracked in git), the same way the Lingui `.po` catalogs are committed.

## The contract: regenerate before committing docs changes

This is a **manual, commit-time** step — deliberately **not** a CI commit-back workflow (that
raced the deploy: deploy built the old `kb/` before the regenerated files were committed).
The model mirrors i18n `.po` files: you regenerate, then commit the output **in the same
commit** as the source change.

```bash
pnpm run generate:agent-kb   # reads docs/content, writes apps/erp/app/modules/agent/kb
```

When to run it:

- You edited anything under `docs/content/**` (added, changed, or removed a doc).
- You changed `scripts/generate-agent-kb.ts` itself.

Then stage the regenerated `apps/erp/app/modules/agent/kb/**` alongside your docs edit.

The **check-and-commit skill runs this automatically** when `docs/content/**` or the
generator is in the change set (see its "agent knowledge base" gate) and includes the
regenerated `kb/` in the commit — so if you commit through that skill, it's handled for you.

## Never

- **Never hand-edit `apps/erp/app/modules/agent/kb/**`** — it's generated; edits are
  overwritten on the next run. Change the source under `docs/content/` instead.
- Never reintroduce a CI workflow that regenerates **and commits** `kb/` on push to `main`
  — it races the deploy build. If you want CI safety, add a **verify-only** check that fails
  when `kb/` is stale (regenerate in CI, then `git diff --exit-code kb/`), without committing.

## How the agent uses it

`apps/erp/app/modules/agent/agent.kb.ts` imports `manifest.json` and bundles the `.md` via
`import.meta.glob`. `search_docs` ranks manifest entries by keyword; `read_doc` returns the
full markdown by its public docs URL (`https://docs.carbon.ms/<slug>`). The agent only ever
sees URLs — slugs/file paths are never surfaced to the user.
