Printing Settings — Layout Approaches

How should the printer assignment hierarchy be displayed?

A

Indented Tree List

A flat list with indentation levels showing the hierarchy. Each row shows the context (location, shipping, receiving, work center) and the assigned printer — or a warning when none is set. Inline editing via dropdowns. Feels like a file tree.

Main Warehouse — Zebra 4x6
Shipping — Zebra 2x1
Receiving — inherits Zebra 4x6
WC: Assembly Line 1 — Zebra 2x1
WC: Pack Station — ⚠ No printer
East Facility — ⚠ No printer
Shipping — ⚠ No printer
Receiving — ⚠ No printer
B

Expandable Location Cards

Each location is a collapsible card. Header shows location name + default printer status. Expand to see shipping, receiving, and work center overrides inside. More visual separation between locations, but takes more vertical space.

▾ Main Warehouse Zebra 4x6
Shipping → Zebra 2x1
Receiving → inherits default
Assembly Line 1 → Zebra 2x1
Pack Station → ⚠ not set
▸ East Facility ⚠ No default
C

Hybrid: Printers List + Assignment Tree

Split into two sections. Top section: simple list of physical printers (add/edit/delete/test). Bottom section: the assignment tree (either A or B style) where you assign those printers to locations and contexts. Separates "what printers exist" from "where are they used."

Assignments
